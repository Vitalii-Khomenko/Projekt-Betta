# Contributing

Thanks for helping improve Betta-Morpho.

## Ground Rules

- keep contributions aligned with authorized security research and defensive use
- do not submit code, signatures, datasets, captures, credentials, or artifacts you do not have the right to share
- avoid introducing code or data with unclear licensing provenance

## Sign-Off Requirement

This project uses the Developer Certificate of Origin in [DCO](DCO).

Every commit submitted for inclusion should be signed off:

```bash
git commit -s -m "Your message"
```

That adds a line like:

```text
Signed-off-by: Your Name <you@example.com>
```

## By Contributing

By submitting a contribution, you represent that:

- you have the right to submit it
- you are not knowingly contributing material you lack permission to share
- your contribution may be distributed under the repository license

## Practical Notes

- keep changes focused and well described
- include tests where practical
- document user-facing behavior changes in `README.md` when relevant
- if your change affects scan behavior, speed, or artifact format, mention that clearly in the PR or commit message
